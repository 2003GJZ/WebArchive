项目名称：WebArchive / 知识库静态化剪辑工具
1. 项目简介
WebArchive 是一款个人知识管理工具，旨在将互联网上零散的高价值内容（如技术博客、在线文档、知识库）转化为永久存储的静态资源。

通过浏览器插件一键抓取，系统会自动处理图片转存、路径替换和样式固化，解决原网页 404、内容被删、图片防盗链或付费墙导致的知识流失问题，构建一个真正属于自己的本地离线知识库。

2. 系统架构
项目分为三个核心端，采用“捕获-处理-展示”的闭环逻辑：

采集端 (Browser Extension)： 负责感知网页内容，进行初步清洗并发送指令。

服务端 (Backend Server)： 负责 HTML 的二次加工、资源本地化下载及数据持久化。

管理端 (Frontend Dashboard)： 负责已归档内容的检索、分类查看和管理。

3. 各端功能详细定义
📦 浏览器插件端 (Collector)
一键抓取： 点击插件图标，自动识别并提取当前页面的正文区域。

Readability 解析： 去除广告、侧边栏、干扰性弹窗，仅保留标题、作者和核心正文。

手动选区 (可选)： 允许用户手动勾选想要抓取的 DOM 节点，适应复杂的网页结构。

元数据采集： 自动记录原始 URL、页面 Title、抓取时间及网站 Favicon。

⚙️ 后端服务端 (Processor & API)
内容流水线： 接收插件传来的 HTML，利用解析库进行深度清洗。

资源本地化 (核心)： * 自动识别 <img>, <link>, <video> 标签。

将远程资源下载到服务器本地目录或对象存储。

链接重写： 将 HTML 中的远程 URL 替换为服务器的相对路径或本地 API 路径。

静态化存储： 生成独立的 .html 静态文件，并配合相应的 CSS 确保样式还原。

API 接口： 提供内容的增删改查、分类管理及搜索接口。

🖥️ 前端展示端 (Viewer)
内容看板： 以卡片或列表形式展示已爬取的页面快照。

内置预览： 在系统内部使用安全沙箱 (iframe) 渲染保存好的静态页面，提供沉浸式阅读。

分类与标签： 支持对归档内容进行手动分类和添加标签。

全局搜索： 基于标题或正文内容进行关键词检索，快速定位知识点。

4. 核心业务流程图
用户在目标网页触发插件。

插件提取 Clean HTML 并 POST 到后端。

后端启动异步任务：

下载所有图片至 /assets/images/。

将 HTML 中 src="http://..." 修改为 src="/assets/images/..."。

将处理后的数据写入数据库。

前端通过 API 获取更新，用户即可离线查阅。
